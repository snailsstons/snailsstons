Admin Guide

Der komplette Workflow – von der Idee bis zum veröffentlichten Heft.

Die Tools im Überblick
🖥️
Admin Studio
Zentrale Schaltstelle. Hefte verwalten, Daten eingeben, exportieren. URL: localhost:8080/admin.html
🔧
MD-Fixer
Storyboard-Dateien für den Pacer vorbereiten. Immer zuerst – vor dem Pacer anwenden. Nur „Panel 1 → 16:9" anhaken.
📊
Smart Pacer
Rhythmus-Analyse. Zeigt Ist-Zustand + Empfehlung. Ändert nichts automatisch – nur Vorschläge. Ziel: Score möglichst hoch, abwechslungsreiche Szenen.
📐
Flex Expander
Panel-Anzahl und Seiten-Anzahl ändern. Min 1 – Max 20 Panels. Min 1 – Max 99 Seiten.
🎨
Prompt Engine
Gemini-Prompts pro Panel generieren. Cover-Prompts automatisch im 9:16 Format. Story + Location DNA werden automatisch befüllt.
1
Storyboard
Idee → strukturierter Plan
→
Pro Seite eine MD-Datei schreiben.
Format: Heft X – Seite Y/Z.md
7 Seiten × bis zu 7 Panels ausformulieren.
→
MD-Fixer auf jede Datei anwenden.
Nur „Panel 1 → 16:9" anhaken → Download.
→
Pacer öffnen → gefixte Datei laden → Score prüfen.
Szenen-Vielfalt ist wichtiger als Layout-Typ.
2
Bilder & Prompts
Gemini Rendering
🖼️
Referenzbilder – immer mitschicken
Upload-Reihenfolge in jedem neuen Chat: Felix → Stella → Archie → Luna → Location

Bei Konsistenz-Verlust sofort neuen Chat öffnen. Nie im selben Chat weiter promten.
→
Prompt Engine öffnen → Heft + Seite + Panel wählen → Prompt kopieren.
→
Prompt in Gemini einfügen + 5 Referenzbilder hochladen.
→
Bild approven → als Panel-Datei speichern.
Datei-Namensschema
heft4/cover.jpg
heft4/4_1_P1.jpg  ← Heft/Seite/Panel
heft4/4_1_P2.jpg
heft4/4_2_P1.jpg
heft4/4_1_audio.mp3  ← Suno Track

⚠️ Alles lowercase – auch Ordnernamen
3
Videos
Grok + YouCut
→
Approved Gemini-Bild als Basis in Grok laden → Video starten.
→
3 × 6s Clips = 18s pro Panel. Format: 9:16 vertikal. Sprache: Deutsch.
→
Bottom Bar immer via YouCut – nie in Grok.
Format: X/7 | Snails Stons × Dragons Chain | Heft N – Titel
4
Daten eintragen
Admin Studio
→
Heft im Admin öffnen → Teaser, Cover Story, Location DNA befüllen.
→
Pro Panel: Story, Dialog (Skript), Location DNA eintragen.
→
SAVE → heftN.js herunterladen → in data/ Ordner ablegen.
🎵
Suno Audio (optional)
Pro Seite einen Suno-Track erstellen.
Dateiname: N_Seite_audio.mp3
z.B. 4_3_audio.mp3 für Heft 4, Seite 3.
5
Deployment
Cloudflare Pages
→
Alle Dateien in ZIP packen → Cloudflare Pages → Upload.
→
In heftN.js "released": true setzen → erst dann erscheint das Heft in der Galerie.
→
Testen: Galerie, Reader, Story-Modal, Audio, Progress-Bar, Share-Button. Auf Handy testen.
✅
Workflow-Checkliste im Admin
Im Admin unter TOOLS → Heft Workflow gibt es eine interaktive Checkliste mit allen 16 Schritten. Jeden Schritt antippen = abhaken. Fortschrittsbalken zeigt wo du bist.
Members verwalten
📝
members.md bearbeiten
Format: token:von:bis
z.B. geheim123:2026-03-01:2026-12-31

Jeder Member bekommt einen eigenen Token. Abgelaufene Tokens werden automatisch abgelehnt.
1
Neue Zeile in members.md eintragen
2
Datei auf Cloudflare deployen
3
Token per Mail an Member schicken
🔒
Heft oder Panel sperren
Ganzes Heft: "members": true in heftN.js

Einzelnes Panel: "bonus": true beim Panel-Eintrag

Members sehen alles – Gäste sehen das Schloss.
Wichtige Regeln
⚠️
Immer lowercase
Ordner und Dateinamen immer kleingeschrieben.
heft4/ ✅   Heft4/ ❌
Cloudflare (Linux) ist case-sensitive – Großschreibung = 404.
💾
Nach jeder Session SAVE drücken
Admin speichert nur im localStorage. Bei Cache-Löschung gehen ungespeicherte Daten verloren. Nach jeder Arbeitssession exportieren.
🔄
Gemini Chat-Limit
Wenn Gemini die Charaktere nicht mehr konsistent rendert → sofort neuen Chat öffnen. Referenzbilder neu hochladen. Nie im selben Chat weiter promten.
